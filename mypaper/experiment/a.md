# 第六章 结论与展望

## 6.1 研究总结

本文始于敏捷软件开发环境下工作量评估面临的严峻挑战：即如何在需求快速演进与信息碎片化的双重约束下，打破传统评估方法在“效率”与“精度”上的零和博弈。针对引言中指出的 GB/T 36964-2018 标准落地难、IFPUG 专家依赖度高以及传统回归模型难以捕捉非线性复杂度等行业痛点，本文构建了“基于大模型思维图谱（GoT）的自动化规模提取”与“基于多智能体强化学习（MARLFS）的动态工作量预测”的双引擎评估框架。通过系统的理论推导与实证研究，本文主要得出以下结论：

### 6.1.1 大模型思维图谱（GoT）实现了软件规模的语义感知自动化
针对引言中提到的**“规模提取效率低、主观性强”**问题，本文的研究证明了大型语言模型（LLM）结合思维图谱范式是解决这一问题的有效途径。
1.  **突破线性推理瓶颈**：实验表明，传统的线性提示工程（如 CoT）在处理复杂的 IFPUG 规则判别时存在显著的逻辑退化。本文提出的 GoT 架构通过显式的状态空间搜索、多视角验证与逻辑聚合机制，成功将隐式的造价规则转化为模型可执行的结构化思维路径。在 DeepSeek-V3 等模型的加持下，GoT 在 ILF/EIF 等关键功能点的判别任务中实现了接近 100% 的准确率，有效消除了人为的主观偏差（Bias）。
2.  **从语法匹配到语义认知**：研究发现，基于 LLM 的评估体系超越了传统 NLP 技术的关键词匹配局限。GoT 架构展现出了强大的“语义感知”能力，能够理解非结构化需求文档中隐含的业务实体与逻辑边界，从而实现了对软件规模的自动化、标准化度量，为敏捷开发中快速获取准确的规模数据提供了坚实的技术支撑。

### 6.1.2 强化学习（MARLFS）重塑了数据驱动的复杂性度量
针对引言中提到的**“工作量预测模型僵化、特征挖掘不足”**问题，本文利用多智能体强化学习技术，重构了从规模到工作量的转化逻辑。
1.  **动态博弈优于静态统计**：通过将特征选择建模为多智能体博弈过程，MARLFS 算法克服了传统 Filter 和 Wrapper 方法在处理高维、非线性软件工程数据时的局限性。在 SEERA 数据集上的实证结果显示，该方法构建的预测模型在均方误差（MSE）指标上显著优于专家估算与传统回归模型，证明了其在捕捉项目复杂非线性关系上的卓越能力。
2.  **管理语义的再发现**：MARLFS 不仅提升了预测精度，更挖掘出了具有深刻管理学意义的关键特征（如“团队凝聚力”、“需求稳定性”等）。这些特征实质上构成了数据驱动的“动态调整因子”，替代了传统方法中主观打分的通用系统特性（GSC），使工作量评估过程具备了更强的可解释性与风险预警能力。

综上所述，本文提出的双引擎框架成功打通了 GB/T 36964-2018 标准中“先规模、后工作量”的数字化全链路。前端利用 AI 大模型解决非结构化输入的标准化理解，后端利用强化学习解决复杂影响因子的动态量化，从而在敏捷开发的高动态环境下，实现了既快速（自动化）又精准（智能化）的工作量评估目标。

## 6.2 研究局限

尽管本文验证了所提框架的有效性，但对照引言中提及的广泛工业应用场景，本研究仍存在以下局限：

1.  **评估范围的完备性不足**：在软件规模估算方面，目前的 GoT 架构主要验证了数据功能点（ILF/EIF）的识别效能。对于涉及复杂时序逻辑与状态流转的事务功能点（EI/EO/EQ），现有的静态图谱结构尚未完全覆盖。此外，面对工业界动辄数百页的超长需求规格说明书，大模型的上下文窗口限制与“中间迷失”现象仍是制约全量自动化落地的技术瓶颈。
2.  **算法的数据依赖与冷启动**：强化学习模型（MARLFS）的表现高度依赖于高质量的历史数据沉淀。对于缺乏历史项目数据积累（Data Scarcity）的初创团队或新兴领域项目，智能体难以通过充分的试错来学习最优策略，可能导致系统在应用初期的表现不如经验丰富的专家直觉。
3.  **计算成本与经济性**：GoT 范式带来的指数级 Token 消耗与强化学习训练所需的算力资源，使得该方法的边际成本远高于传统的类比法。在缺乏自适应选择机制的情况下，对于低风险的小型敏捷迭代任务，全量部署该系统的投入产出比（ROI）面临挑战。

## 6.3 未来展望

基于上述结论与局限，未来的研究将致力于构建更具适应性、更经济且更可信的下一代智能评估体系，以进一步回应引言中提出的数字化转型挑战：

1.  **构建全覆盖的动态思维图谱**：将图谱提示工程扩展至全类型功能点评估，设计包含时序逻辑节点的动态 GoT 架构，以实现对事务功能点的精准识别。同时，引入检索增强生成（RAG）技术，在大模型外部挂载行业标准库与历史案例库，以增强模型处理长文档与领域特定知识的能力。
2.  **探索迁移学习与元学习框架**：为解决强化学习的冷启动问题，未来将研究基于迁移学习（Transfer Learning）的特征选择机制。利用大型公开数据集（如 ISBSG）预训练智能体，使其掌握通用的软件工程特征交互规律，再迁移至特定组织的小规模数据集进行微调，从而降低对单一项目数据的依赖。
3.  **研发自适应的“元推理”机制**：为了平衡精度与成本，将构建一种自适应路由系统。该系统能够根据输入任务的文本特征与逻辑复杂度，动态选择最经济的推理范式（如简单任务降级为 CoT，复杂任务升级为 GoT），实现计算资源的按需分配。
4.  **人机协同生态的深度融合**：最终，本研究旨在推动评估工具从“离线预测”向“在线决策”演进。未来将探索把 MARLFS 与 GoT 引擎嵌入 DevOps 流水线与项目管理工具（如 Jira），利用持续产生的过程数据进行在线学习，构建“AI 实时审计 + 专家关键决策”的人机协同新生态，为软件工程管理的智能化转型提供持续动力。
